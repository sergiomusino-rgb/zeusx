## Struttura tabella app_registry

| Campo | Tipo | Descrizione |
|-------|------|-------------|
| id | UUID (PK) | Identificativo unico |
| reseller_id | UUID | Riferimento a auth.users (rivenditore) |
| app_name | TEXT | Nome dell'app |
| app_url | TEXT | URL pubblico (/a/:slug) |
| status | TEXT | Stato: 'active', 'suspended', 'pending' |
| monthly_fee | DECIMAL(10,2) | Quota mensile in EUR |
| zeusx_share | DECIMAL(10,2) | Percentuale ZEUSX (es. 25.00 = 25%) |
| created_at | TIMESTAMPTZ | Data creazione |
| updated_at | TIMESTAMPTZ | Data aggiornamento |

## Funzioni SQL disponibili

### 1. get_reseller_apps(p_reseller_id UUID)
Estrae l'elenco delle app per un rivenditore specifico.

```sql
SELECT * FROM get_reseller_apps('uuid-del-rivenditore');
```

### 2. get_zeusx_total_due(p_reseller_id UUID)
Calcola il totale dovuto a ZEUSX sommando le percentuali di tutte le app attive.

```sql
SELECT get_zeusx_total_due('uuid-del-rivenditore');
-- Ritorna: 125.50 (esempio)
```

### 3. get_reseller_apps_with_total(p_reseller_id UUID)
Vista combinata: elenco app + totale ZEUSX in un'unica chiamata.

```sql
SELECT * FROM get_reseller_apps_with_total('uuid-del-rivenditore');
```

## API REST disponibili

| Metodo | Endpoint | Descrizione |
|--------|----------|-------------|
| GET | /api/app-registry | Lista app con totale ZEUSX |
| GET | /api/app-registry/:id | Dettaglio singola app |
| POST | /api/app-registry | Registra nuova app |
| PATCH | /api/app-registry/:id | Aggiorna app |
| DELETE | /api/app-registry/:id | Rimuovi app |

## Esempio di utilizzo

```javascript
// Registrare una nuova app
const response = await fetch('/api/app-registry', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    app_name: 'Mia App Cliente',
    app_url: 'https://zeusx.com/a/mia-app',
    monthly_fee: 49.00,
    zeusx_share: 25.00,
    status: 'active'
  })
});

// Ottenere elenco app con totale
const { apps, total_zeusx_due } = await fetch('/api/app-registry', {
  headers: { 'Authorization': `Bearer ${token}` }
}).then(r => r.json());
```

## Note
- La tabella ha RLS abilitato
- I rivenditori vedono solo le loro app
- La funzione `get_zeusx_total_due` calcola solo le app con status='active'
- Non modifica alcuna tabella esistente
