# Risoluzione problema lettura app "pizzeria"

## Problema
L'API `/api/test` funziona (legge per `totalum_app_id`), ma l'app "pizzeria" non viene letta correttamente.

## Cause possibili

### 1. Policy RLS mancanti
La tabella `apps` potrebbe non avere le policy RLS corrette per permettere la lettura tramite `totalum_app_id`.

**Soluzione:** Eseguire la migrazione `20260720_fix_apps_rls_for_totalum_app_id.sql`

### 2. Record app non esiste o ha valori errati
L'app potrebbe non esistere nel database, o potrebbe avere:
- `totalum_app_id` diverso da "pizzeria" (spazi bianchi, caratteri invisibili)
- `client_active = false`
- `slug` diverso da "pizzeria"

**Soluzione:** Eseguire lo script `backend/check_pizzeria_app.js` per verificare

### 3. API cerca solo per slug
L'API `/a/:slug` cercava solo per `slug`, non per `totalum_app_id`.

**Soluzione:** Modificata `backend/routes/client-app.js` per supportare entrambi i casi

## Istruzioni per la risoluzione

### Passo 1: Verificare i dati esistenti
```bash
cd backend
node check_pizzeria_app.js
```

### Passo 2: Creare l'app se non esiste
```bash
cd backend
node create_pizzeria_app.js
```

### Passo 3: Applicare le policy RLS
Eseguire nel database Supabase:
```sql
-- Dal file 20260720_fix_apps_rls_for_totalum_app_id.sql
ALTER TABLE IF EXISTS apps ENABLE ROW LEVEL SECURITY;

DROP POLICY IF EXISTS "apps_select_by_totalum_app_id" ON apps;
CREATE POLICY "apps_select_by_totalum_app_id" ON apps 
    FOR SELECT 
    USING (client_active = true);

DROP POLICY IF EXISTS "apps_service_role_all" ON apps;
CREATE POLICY "apps_service_role_all" ON apps 
    FOR ALL 
    USING (auth.role() = 'service_role')
    WITH CHECK (auth.role() = 'service_role');

CREATE INDEX IF NOT EXISTS idx_apps_totalum_app_id ON apps(totalum_app_id);
```

### Passo 4: Riavviare il backend
```bash
cd backend
npm restart
```

## Verifica
Testare l'API:
```bash
curl -X GET "http://localhost:3000/api/test?totalum_app_id=pizzeria"
```

Se l'app esiste e `client_active = true`, dovrebbe restituire i dati dell'app.
