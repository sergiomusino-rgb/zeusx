# ZeusX - Slot Cumulative Model Migration

## Data: 2026-07-22

## Descrizione

Questa migrazione implementa il modello cumulativo degli slot per i piani di abbonamento.

## Regole Implementate

1. **Nessun piano (Free)**: 0 slot di base
2. **Piano Starter (4,99€)**: +1 slot
3. **Piano Pro**: +5 slot (cumulativo)
4. **Piano Business (250€)**: +100 slot

## Modifiche al Database

### Tabella `tenants`

- `app_limit`: Slot totali disponibili (somma cumulativa di tutti gli slot acquistati)
- `total_apps_created`: Contatore permanente di tutte le app create (non decrementa mai)

### Funzioni SQL

- `add_tenant_slots(tenant_id, slots_to_add)`: Aggiunge slot in modo cumulativo
- `get_tenant_slots_available(tenant_id)`: Restituisce gli slot disponibili

## Modifiche al Codice

### File Modificati

1. **frontend/app/api/tenants/create/route.ts**
   - Nuovi utenti: `plan: 'free'`, `app_limit: 0`
   - Profilo: `subscription_plan: 'free'`

2. **frontend/app/api/apps/route.ts**
   - `getOrCreateTenant`: `app_limit: 0` per nuovi tenant
   - `canCreateApp`: Admin ha app illimitate, gli altri hanno controllo slot

3. **frontend/app/api/generate/route.ts**
   - `getUserTenant`: Crea tenant con `app_limit: 0` se non esiste
   - Admin: salta il controllo degli slot

4. **frontend/app/actions/generator.ts**
   - Aggiunto controllo slot prima della creazione dell'app
   - Admin: salta il controllo degli slot

5. **frontend/app/api/webhooks/stripe/route.ts**
   - `getSlotsForPlan`: Mappa piani agli slot (già corretto)
   - Aggiunge slot in modo cumulativo con `app_limit + slotsToAdd`

## Come Funziona il Modello Cumulativo

Quando un utente acquista un piano:
1. Il webhook Stripe riceve l'evento `checkout.session.completed`
2. Estrae `plan_id` e `tenant_id` dai metadata
3. Chiama `getSlotsForPlan(planId)` per ottenere gli slot del piano
4. Aggiorna `app_limit` aggiungendo gli slot in modo cumulativo: `app_limit = app_limit + slotsToAdd`

### Esempio

- Utente senza piano: 0 slot
- Acquista Starter: 0 + 1 = 1 slot
- Passa a Pro: 1 + 5 = 6 slot totali
- Passa a Business: 6 + 100 = 106 slot totali

## Esecuzione della Migrazione

Eseguire il file SQL `20260722_update_tenants_slots_cumulative.sql` sul database Supabase per aggiornare i tenant esistenti.
